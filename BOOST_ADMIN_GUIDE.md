## 五、审核流程说明

### 用户发布任务流程：
```
1. 用户在前端提交任务
   ↓
2. 系统冻结用户资金（任务金额 + 押金）
   ↓
3. 任务状态为"待审核"
   ↓
4. 管理员在后台审核
   ↓
5a. 审核通过 → 任务发布，资金实际扣除
5b. 审核拒绝 → 任务取消，资金退还用户
```

### 资金处理逻辑：
- **发布时**: 冻结资金（money 减少，freeze_money 增加）
- **审核通过**: 实际扣除（保持冻结状态）
- **审核拒绝**: 退还资金（money 增加，freeze_money 减少）

## 六、新增数据库字段说明

### User 表需要添加字段：
```sql
ALTER TABLE `fa_user` 
ADD COLUMN `freeze_money` decimal(10,2) NOT NULL DEFAULT '0.00' COMMENT '冻结金额' AFTER `money`;
```

## 八、安全建议

1. **权限控制**
   - 只给管理员分配审核权限
   - 普通客服人员只给查看权限

2. **审核规范**
   - 制定明确的审核标准
   - 记录审核备注信息
   - 定期抽查已审核任务

3. **资金安全**
   - 确保冻结和退还逻辑正确
   - 定期检查资金流水
   - 设置资金变动通知

## 九、常见问题

### Q1: 用户发布任务后为什么看不到？
A: 因为任务处于"待审核"状态，需要管理员审核通过后才会显示在列表中。

### Q2: 审核拒绝后资金如何退还？
A: 系统会自动将冻结的资金退还给用户，包括任务金额和押金。

### Q3: 如何查看审核记录？
A: 在任务列表中可以看到每个任务的审核状态，在任务详情中可以看到审核时间和审核管理员。

### Q4: 可以批量审核吗？
A: 可以，在任务列表中勾选多个任务，然后点击"审核通过"或"审核拒绝"按钮。
